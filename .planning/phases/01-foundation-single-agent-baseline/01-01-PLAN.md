  </read_first>
  <action>
    Create SKELETON.md at the project root using the Walking Skeleton template. Document these architectural decisions:

    - Framework: Vanilla PHP (no Laravel/Symfony). Rationale: zero framework overhead, all needed capabilities built into PHP 8.5.4.
    - Dependency strategy: Zero external dependencies for Phase 1. No Composer. PHP extensions only (curl, json, mbstring, readline, pcntl, posix, sockets).
    - Config format: JSON default (config.json, preferences.json) per D-01. PHP arrays also accepted. Loader detects format by file extension (.json vs .php).
    - Autoloading: PSR-4-like via spl_autoload_register. App\ namespace maps to src/.
    - Logging: Custom Logger class (~70 lines). Human-readable formatted text with channel prefix ([SYSTEM], [AGENT]) per D-11, D-12. Single application log file for Phase 1 per D-13.
    - LLM access: Single LlmClient with provider adapter pattern per D-04. PHP curl directly per D-05. Provider config specifies provider, model, api_key per D-06.
    - CLI entry point: Flat script research.php at project root per D-07. Direct question argument only per D-08. Interactive REPL deferred to Phase 5.
    - SOUL.md format: Structured markdown sections per D-09: ### Identity, ### Values, ### Instructions, ### Constraints.
    - Logging format per D-11: [timestamp] [CHANNEL] [LEVEL] [correlation_id] message {context}.
    - Directory layout: src/ organized by domain (Config/, Log/, LlmClient/, Agent/), config/ with per-agent directories, logs/ at project root, bin/ for utility scripts.

    Create all directories by running: mkdir -p src/Config src/Log src/LlmClient src/Agent config/agents/researcher config/arbitrator logs bin

    The SKELETON.md Capability end-to-end statement should read: "A user can run `php research.php "question"` from the command line and receive a research answer produced by an LLM agent configured with a provider model, API key, and personality."
  </action>
