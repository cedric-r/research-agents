  <action>
    Create src/bootstrap.php with a PSR-4-like autoloader:
    - Register spl_autoload_register callback
    - Map App\ namespace prefix to src/ directory
    - Convert class name to file path: App\Config\Loader -> src/Config/Loader.php
    - Use require (not require_once) for performance — autoloader guarantees each class is registered once
    - Use file_exists before require to avoid missing-file warning
    - Set explicit error reporting at top: error_reporting(E_ALL), ini_set('display_errors', '1'), ini_set('log_errors', '1')

    Create src/Config/ConfigException.php:
    - Namespace: App\Config
    - Extend \RuntimeException
    - No additional methods — just a typed catch-target for config errors
    - Per D-02, PHP 8.5.4 means modern PHP features are available (typed properties, match expressions)

    Create src/Config/Loader.php:
    - Namespace: App\Config
    - Single public method: load(string $path, array $required = [], array $types = []): array
    - Flow per RESEARCH.md Pattern 1:
      1. Check file_exists — throw ConfigException if not found with path in message
      2. Detect format by file extension (.json vs .php)
      3. For .json: file_get_contents -> json_decode($raw, true, 32, JSON_THROW_ON_ERROR) — per D-01, D-02, RESEARCH Pitfall 1
      4. For .php: include — per D-02, PHP arrays loaded via include. Guard: only load .php files from within project config directory (not user-provided paths). Result must be array — throw ConfigException if not.
      5. Aggregate validation: collect all missing fields first. Per RESEARCH Anti-Pattern: "Always validate all required fields and report every error in one message."
      6. Validate types after decode: use gettype() to check types against $types map
      7. Non-empty string check for string-type fields (trim($value) !== '')
      8. If errors accumulated, throw ConfigException with ALL errors joined by newline
      9. Return validated config array
    - Per Pitfall 5: errors must be clear and actionable. Format: "Config validation failed for {path}:\n  - Missing required field: 'api_key'\n  - Field 'model' must be a non-empty string"
    - Per Security Domain: never include the full decoded config in error messages (API key could leak). Use field names only, not values.
    - Handle PHP array include carefully: scope the include by extracting a closure that returns the value, so the included file's variables don't leak
  </action>
  <acceptance_criteria>
    - src/bootstrap.php contains spl_autoload_register and maps App\ to src/
    - src/Config/Loader.php loads valid JSON configs and returns array

    - src/Config/Loader.php throws ConfigException for missing required fields with ALL field names listed

  </read_first>
  <action>
    Create src/Log/Logger.php:
    - Namespace: App\Log
    - Constructor: __construct(string $file, string $channel = 'SYSTEM', ?string $correlationId = null)
      - Channel truncated to 11 chars with mb_substr, uppercased
      - Correlation ID: use provided or generate via self::generateCorrelationId() which uses bin2hex(random_bytes(4)) for 8-char hex string per RESEARCH
    - Core method: log(string $level, string $message, array $context = []): void
      - Format per D-11: [2026-06-13 14:30:00.123456] [CHANNEL] [LEVEL] [correlation_id] message {"json": "context"}
      - Use (new \DateTimeImmutable())->format('Y-m-d H:i:s.u') for microsecond timestamps
      - Channel field left-aligned in 11-char column for alignment
      - Level field left-aligned in 5-char column: sprintf("[%-11s] [%-5s]", $channel, $level)
      - Context json-encoded with JSON_UNESCAPED_SLASHES | JSON_UNESCAPED_UNICODE
      - Write with file_put_contents($this->file, ..., FILE_APPEND | LOCK_EX)
      - Per RESEARCH Don't Hand-Roll: LOCK_EX prevents log line interleaving between concurrent processes
      - Suppress write errors with @ and fall back to error_log()
      - Per Security Domain: strip control characters from message: preg_replace('/[\x00-\x08\x0B\x0C\x0E-\x1F]/', '', $message)
    - Convenience methods: info(), error(), warn(), debug() — each calls log() with the appropriate level string
    - Static utility: generateCorrelationId(): string — returns 8-char hex string
    - Levels const array: ['DEBUG', 'INFO', 'WARN', 'ERROR'] — validate $level against this list
    - Per D-12: channel parameter on constructor allows different logger instances for different channels (but all write to the same file with channel prefix)
    - Per D-13: single log file (no per-session files in Phase 1)
    - Per Pitfall 5: use file_put_contents which handles atomic writes
    - Ensure logs/ directory exists: auto-create with is_dir/mkdir if the log file's directory doesn't exist yet
  </action>
  <acceptance_criteria>
    - Logger constructor accepts file path, channel name, and optional correlation ID
    - log() writes formatted line to the specified file with timestamp, channel, level, correlation ID, message
    - Concurrent log writes do not interleave lines (LOCK_EX)
    - info(), error(), warn(), debug() convenience methods produce correct level strings
    - generateCorrelationId() returns 8-character hex string
    - Log directory is auto-created if it doesn't exist

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system -> Config Loader | Config files read from disk. Only known local files accessed — no remote load. |
| User -> Config Loader | Config file paths are hardcoded, not user-provided. No arbitrary file read. |
| Logger -> Log file | Log entries written to file. LOCK_EX prevents interleaving between concurrent processes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Information Disclosure | Config Loader error messages | mitigate | Never include API key or decoded config values in exception messages. Reference field names only. |
| T-01-04 | Tampering | PHP array include | mitigate | Only include .php files from known project config paths. File paths are hardcoded, not user-supplied. Use closure-scoped include to prevent variable leakage. |
| T-01-05 | Tampering | Log file injection | mitigate | Strip control characters (\x00-\x08\x0B\x0C\x0E-\x1F) from log messages using preg_replace before writing. |
| T-01-06 | Spoofing | json_decode null return | mitigate | Always use JSON_THROW_ON_ERROR flag per D-02 and RESEARCH Anti-Pattern. Never surface json_last_error silently. |
</threat_model>
