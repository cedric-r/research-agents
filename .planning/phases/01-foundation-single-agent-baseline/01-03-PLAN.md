    - Namespace: App\LlmClient
    - Extend \RuntimeException
    - Typed catch-target for LLM-specific errors (network failure, API error, bad response)

    Create src/LlmClient/LlmClient.php:
    - Namespace: App\LlmClient
    - Constructor: __construct(array $providerConfig)
      - Store: baseUrl (rtrim with /), apiKey, model
      - Read from $providerConfig keys: 'base_url', 'api_key', 'model', 'provider' (for potential provider-specific behavior)
    - Single public method: chat(array $messages, array $options = []): string
    - Flow:
      1. Build payload: merge model, messages, temperature (0.7 default), max_tokens (4096 default) with $options override
      2. curl_init with POST to $this->baseUrl . '/chat/completions'
      3. curl_setopt_array with: CURLOPT_POST, CURLOPT_POSTFIELDS (json_encode payload), CURLOPT_HTTPHEADER (Content-Type: application/json, Authorization: Bearer), CURLOPT_RETURNTRANSFER, CURLOPT_TIMEOUT (60 default, overridable from $options), CURLOPT_CONNECTTIMEOUT (10)
      4. curl_exec — MUST check for false BEFORE any other processing (Pitfall 3: curl_exec returns false on error, not empty string)
      5. curl_errno + curl_error for diagnostic info if exec fails
      6. curl_getinfo for HTTP status code
      7. curl_close
      8. If $response is false: throw LlmException with curl error message
      9. If HTTP status !== 200: throw LlmException with status code and truncated response body (first 500 chars per RESEARCH Pattern 2)
      10. json_decode($result, true, 32, JSON_THROW_ON_ERROR) — must be inside try/catch to handle malformed JSON responses (Pitfall 2)
      11. Extract content from $result['choices'][0]['message']['content'] ?? ''
      12. Return the content string
    - Per D-04: provider differences (base URL, auth header format, response shapes) handled via config — no separate classes per provider
    - Per Security Domain: never include full API request/response in error messages. Redact Authorization header value in logs. Truncate response to 500 chars.
    - Per D-15: the LlmClient should return usage stats alongside the answer for metadata display. Return an array with 'content', 'model', 'usage' keys instead of just string — or have ResearchAgent extract from raw response.
      Decision per Claude's discretion: Return content as string. ResearchAgent handles metadata extraction from raw response via a separate method: getLastResponseMetadata(): array
    - Add a private property $lastResponse to store the decoded response for metadata retrieval
    - Add method: getLastResponseInfo(): array — returns ['model' => ..., 'usage' => [...], 'response_time_ms' => ...]
  </action>
  <acceptance_criteria>
    - LlmClient::chat() sends POST to {baseUrl}/chat/completions with Bearer auth
    - Successful response returns content string
    - curl_exec failure throws LlmException with curl error message
    - HTTP non-200 throws LlmException with status code
    - Malformed JSON response throws LlmException
    - getLastResponseInfo() returns array with model, usage keys after a successful call
  </acceptance_criteria>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> CLI | Research question comes from CLI argument. Length-capped, never eval'd. |
| CLI -> Provider API | HTTP request to external LLM provider. API key transmitted via Authorization header. |
| Provider API -> CLI | Untrusted response content parsed and displayed. Response is text content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Information Disclosure | curl error messages | mitigate | Never include full request body or Authorization header in error messages. Truncate API response in error messages to 500 chars. |
| T-01-08 | Denial of Service | LLM API timeout | mitigate | CURLOPT_TIMEOUT=60s, CURLOPT_CONNECTTIMEOUT=10s enforced on all HTTP calls. curl_exec false check before any processing. |
| T-01-09 | Elevation of Privilege | CLI input injection | mitigate | Research question is sent as user message to LLM. Never passed to shell, eval, or include. Capped at 2000 chars. |
| T-01-10 | Spoofing | curl_exec false masking | mitigate | Always check curl_exec return === false before json_decode. RESEARCH Pitfall 3: false is masked by null when feeding to json_decode. |
| T-01-11 | Information Disclosure | Question in log output | mitigate | Logged question truncated to 200 characters before writing to log file per Security Domain V8 guidelines. |
</threat_model>
