## 3. CLI REPL Contract

### 3.1 Prompt Format

| Element | Format | Source |
|---------|--------|--------|
| Prompt | `\x01\e[36m\x02research> \x01\e[0m\x02` (bold cyan) | CONTEXT.md D-03 + Pitfall 1 |
| Startup message | `"ResearchAgents REPL -- {N} agents configured, ready."` | CONTEXT.md Specific Ideas |
| Command prefix | `/` | CONTEXT.md D-02 |

The `\x01`/`\x02` wrapping around ANSI codes is **mandatory** when used in `readline()` prompts to prevent cursor-position miscounting (RESEARCH.md Pitfall 1).

### 3.2 ANSI Color Palette (CLI)

| Role | ANSI Code | Readline-Safe Wrapper | Usage |
|------|-----------|-----------------------|-------|
| Foreground green | `\e[32m` | `\x01\e[32m\x02` | Winner name (with bold), high scores (>=8) |
| Foreground cyan | `\e[36m` | `\x01\e[36m\x02` | Section headers (with bold), prompt |
| Foreground yellow | `\e[33m` | `\x01\e[33m\x02` | Mid-range scores (5-7), command highlights |
| Foreground red | `\e[31m` | `\x01\e[31m\x02` | Low scores (<5), error messages (with dim) |
| Foreground white | `\e[37m` | `\x01\e[37m\x02` | Body text |
| Bold | `\e[1m` | `\x01\e[1m\x02` | Section headers, winner name, command names |
| Dim | `\e[2m` | `\x01\e[2m\x02` | Secondary info, errors (with red), separators |
| Reset | `\e[0m` | `\x01\e[0m\x02` | End every styled segment |

| Element | ANSI Treatment | Source |

### 3.4 Score Table Format (CLI)

Existing `research.php` output format reused verbatim:

```
 ─────────────────────────────────────────────────────────────
  SCORE TABLE
 ─────────────────────────────────────────────────────────────
  Agent              Quality  Critique  Diversity  Total
 ─────────────────────────────────────────────────────────────
  alpha *               8.5/10     0.75      0.12    0.825
  beta                  6.2/10     0.60      0.00    0.510 << WINNER
 ─────────────────────────────────────────────────────────────
 * Candidate for final selection
```

- 18-character left-aligned agent name column
- 6-character right-aligned Quality column with `/10` suffix
- 7-character right-aligned Critique, Diversity, Total columns
- `*` marker for candidates, `<< WINNER` marker for winner
- Column headers: Bold Cyan
- Score values: colored by threshold (green/yellow/red)
- Winner name next to score: Bold Green

### 3.5 Command Output Formats

| Command | Output Format |
|---------|--------------|
| `/help` | Lists commands. Each command line: `Bold Yellow /command` + `: description` in reset. |
| `/replay <slug>` | Reads `sessions/{slug}/session.md`, renders with ANSI: frontmatter ignored, ## headers in Bold Cyan, **Winner:** in Bold Green, **score_table:** rendered as table with colored values. Errors in Dim Red. |
| `/config` | Lists arbitrator config keys in `Dim key` `: value` format. |
| `/agents` | Lists agent names in their cycled colors, with model in Dim. |
| `/last` | Re-renders the debate output of the most recent session in full detail (same format as after research completes). |
| `/history` | Lists past questions from readline history file. Each line: `{N}. question` where N is the sequential number. |
| `/clear` | Analogous to `clear` shell command: `\e[2J\e[H` ANSI sequence. |
| `/exit` | No output (exits REPL). |

### 3.6 Progress Display (CLI)

During research execution, the REPL shows:

```
⏳ Researching... (2/3 agents)
  [alpha] llm_call — building tool context
  [beta] web_search — searching Brave API
  [alpha] completed
```

- Spinner rotates through `|/-\` at 200ms intervals (RESEARCH.md spinner pattern)
- Agent names displayed in their cycled color
- Event types shown in reset text
- Progress events read from `sessions/{slug}/session.log`
- Spinner line cleared on completion: `\r\e[K`

### 3.7 REPL Lifecycle

| State | Visual |
|-------|--------|
| Startup | `Bold Cyan ResearchAgents REPL — {N} agents configured, ready.` |
| Idle | `Cyan research> ` prompt, blinking cursor |
| Researching | Spinner + log lines from session.log |
| Complete (success) | Full debate output rendered with ANSI colors |
| Complete (failure) | `Dim Red Error: {message}` |
| SIGINT (CTRL+C) | `Yellow Interrupted.` on new line, return to prompt |
| CTRL+D / `/exit` | Newline, exit with history saved |

## 4. Web REPL Contract

### 4.1 Page Structure

| Route | Page | Content |
|-------|------|---------|
| `GET /` | Home | Research question form + link to sessions list |
| `POST /api/ask` | Redirect | Validates question, creates session, `exec()` background process, redirects to `/session/{id}` |
| `GET /session/{id}` | Session View | Page title with question, SSE-connected log area, link back to home |

### 4.6 SSE Format

```
event: progress
data: {"ts":"2026-06-13T10:30:00.123Z","agent":"alpha","event":"started","data":{"question":"...","model":"deepseek-v4-flash"}}

event: done
data: {}

: keepalive

```

- `event: progress` -- each new line from session.log as the `data` field
- `event: done` -- sent when research process completes (checked via PID file + posix_kill)
- `: keepalive` -- comment line sent every 15 seconds to prevent proxy timeout
- Content-Type: `text/event-stream`
- Cache-Control: `no-cache`
- Connection: `keep-alive`
- X-Accel-Buffering: `no` (for Nginx compatibility)

### 4.7 JavaScript Contract (Web)

**Minimal vanilla JS only.** No framework, no bundler, no npm.

The SSE stream handler is a single `<script>` block in the session view page:

```
- Create EventSource to /session/{id}/stream
- On event `progress`: parse JSON from event.data, create a <div class="log-line">, set textContent to "{agent}: {event}" (sanitized), append to log container, auto-scroll to bottom
- On event `done`: close EventSource, display completion message
- On error: retry connection (EventSource does this automatically), show stale indicator after 30s without events
```

**Security:** All user/session content rendered via `.textContent` (not `.innerHTML`) to prevent XSS. LLM output displayed in `<pre>` or text nodes only.

## 5. Session File Contract (Markdown)

### 5.1 File Structure

```
sessions/
├── .gitkeep
└── 2026-06-13_what-are-latest-advances-in-transformer-architectures/
    ├── session.md          # Full transcript
    └── session.log         # Progress events (JSON lines)
```

`session.log` is also consumed by the web SSE endpoint and CLI progress display.

### 5.2 Session Markdown Schema

**YAML frontmatter** (between `---` delimiters):

```yaml
---
question: "What are the latest advances in transformer architectures?"
date: "2026-06-13T10:30:00+00:00"
agent_count: 3
agents: [alpha, beta, gamma]
winner: alpha
correlation_id: a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6
duration_ms: 45000
model_info: "deepseek-v4-flash / claude-sonnet-4 / gpt-4o"
score_summary: "alpha: 0.825, beta: 0.510, gamma: 0.325"
---
```

**Body sections** (mandatory, in this order):

| Section | Heading | Content |
|---------|---------|---------|
| Summary | `## Summary` | Question restated, winner name, duration, score table |
| Raw Answers | `## Raw Answers` | Each agent's full output with metadata |
| Debate | `## Debate` | Winner narrative, judge's reasoning |
| Errors | `## Errors` | Any agent errors or timeouts |

**Score table format** in Summary section:

```markdown
| Agent | Quality | Critique | Diversity | Total |
|-------|---------|----------|-----------|-------|
| alpha | 8.5/10 | 0.75 | 0.12 | 0.825 |
| beta  | 6.2/10 | 0.60 | 0.00 | 0.510 |
```

**Raw Answer section** per agent:

```markdown
### alpha

- **Model:** deepseek-v4-flash
- **Response time:** 12345ms
- **Tokens:** 500 in / 1200 out

{full answer text}

---
```

**Debate section:**

```markdown
## Debate

### Winner: alpha

{judge narrative}
```

**Errors section** (omitted if no errors):

```markdown
## Errors

- **beta:** Timed out after 60s
- **gamma:** LLM API returned 429 rate limit
```

### 5.3 Progress Log Format

One JSON object per line, written by child processes during research:

```
{"ts":"2026-06-13T10:30:00.123Z","agent":"alpha","channel":"PROGRESS","event":"started","data":{"question":"...","model":"deepseek-v4-flash"}}
```

**Event types and when they fire:**

| Event | Who Fires | When |
|-------|-----------|------|
| `started` | Agent child | Agent begins research execution |
| `llm_call` | Agent child | LLM API call for tool context building |
| `web_search` | Agent child | Web search tool executing |
| `paper_search` | Agent child | Paper search tool executing |
| `tool_result` | Agent child | Tool returned results |
| `score_evaluated` | Arbitrator parent | Scoring LLM returned quality scores |
| `critique_r2_started` | Agent child | Round 2 critique prompt sent to LLM |
| `critique_completed` | Agent child | Round 2 critique received |
| `completed` | Agent child | Agent finished successfully |
| `timed_out` | Agent child | Agent exceeded deadline |
| `failed` | Agent child | Agent encountered error |
| `batch_complete` | Arbitrator parent | All agents in a batch completed |

Written via `file_put_contents($logFile, $json . "\n", FILE_APPEND | LOCK_EX)` for atomic concurrent-write safety (RESEARCH.md Pitfall 4).

## 7. Session File Naming

**Pattern:** `{date}_{slug}/session.md` where:

- `date`: `date('Y-m-d')` format (e.g., `2026-06-13`)
- `slug`: First 60 chars of question, lowercased, spaces to hyphens, non-alphanumeric stripped

**Example:**
> Input: `"What are the latest advances in transformer architectures for NLP?"`
> Slug: `what-are-the-latest-advances-in-transformer-architectures`
> Path: `sessions/2026-06-13_what-are-the-latest-advances-in-transformer-architectures/session.md`

Implementation from RESEARCH.md:

```php
function slugFromQuestion(string $question): string
{
    $slug = preg_replace('/[^a-zA-Z0-9-]+/', '-', strtolower($question));
    $slug = trim($slug, '-');
    return rtrim(mb_substr($slug, 0, 60), '-');
}
```

## 9. Security Contract

| Surface | Mitigation | Source |
|---------|------------|--------|
| Command injection (web) | `escapeshellarg()` on question before `exec()` | RESEARCH.md V12 |
| Path traversal (web) | `basename()` on session ID, reject `..` or `/` | RESEARCH.md V5 |
| HTML injection (web) | `htmlspecialchars()` on all session data rendered in HTML | RESEARCH.md V5 |
| XSS in SSE | `.textContent` not `.innerHTML` in JS handler | RESEARCH.md |
| ANSI injection (CLI) | Strip ANSI escape codes from LLM output displayed in terminal | RESEARCH.md V8 |
| Session file access | `sessions/` outside web document root (`public/`) | RESEARCH.md Security Note |
| Network exposure | Web server binds to `localhost` only, not `0.0.0.0` | RESEARCH.md Security Note |

## 10. Anti-Patterns to Avoid

| Anti-Pattern | Why | Correct Approach |
|--------------|-----|------------------|
| Unwrapped ANSI in readline() prompt | Cursor positioning breaks | Wrap in `\x01`/`\x02` |
| Buffered SSE output | Events arrive in batches | `ob_implicit_flush(true)`, `ob_end_clean()`, `output_buffering=off` |
| exec() without output redirect | Web request blocks | `exec("nohup php ... > /path/to/output.log 2>&1 &")` |
| Concurrent writes without LOCK_EX | Log lines interleave/corrupt | `FILE_APPEND | LOCK_EX` on every file_put_contents call |
| CTRL+D as empty string | readline() returns `false` on EOF | Check `$input === false` before `trim()` |
| Relative history file path | History lost across CWD changes | Absolute path via `getenv('HOME')` |
