<objective>
**As a** user, **I want to** submit a research question and receive independent answers from all my configured agents running in parallel, **so that** I can benefit from multiple AI perspectives without waiting sequentially for each agent.

This plan creates the core Arbitrator class with pcntl_fork-based parallel execution, temp file IPC for inter-process result transfer, and basic result collection. It adds the CURLOPT_NOSIGNAL flag to HttpHelper for fork-safe curl operation, trims AgentManager to agent-discovery-only, and wires the Arbitrator into research.php.

**Purpose:** Deliver the fundamental value of Phase 3 -- parallel agent execution with process isolation.

      $running = $children;

          foreach ($running as $pid => $info) {
              $reaped = pcntl_waitpid($pid, $status, WNOHANG);
              if ($reaped > 0 || $reaped === -1) {
                  // Process result from temp file
                  $unset($running[$pid]);

          if (!empty($running)) {
              usleep(100000); // 100ms poll interval
          }
      }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parent process -> child process | fork() creates child with inherited signal handlers, file descriptors, and memory state |
| child process -> filesystem | child writes JSON result to sys_get_temp_dir() temp file |
| filesystem -> parent process | parent reads temp file to collect agent result |
| child process -> external APIs | child makes HTTP calls to LLM/search providers via curl |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Temp file IPC (agent name in path) | mitigate | Sanitize agent names in temp file path: preg_replace('/[^a-zA-Z0-9_-]/', '_', $name). Agent names come from config files (not user input), but defense-in-depth. |
| T-03-02 | Tampering | Temp file data integrity | mitigate | Child writes with LOCK_EX on file_put_contents. Parent reads after child exits -- no concurrent access. PID in filename prevents collision: agent_{name}_{correlationId}_{pid}.json. |
| T-03-03 | DoS | Fork bomb (runaway fork loop) | mitigate | Cap at max_concurrent_agents (default 5). Arbitrator never forks more than limit. Fork failure (pid === -1) is caught and logged, loop continues. |
| T-03-04 | DoS | Zombie processes | mitigate | Every forked child is reaped via pcntl_waitpid in the wait loop. After batch complete, all PIDs are accounted for. SIGKILL path also calls pcntl_waitpid to reap. |
| T-03-05 | Information Disclosure | Forked child inherits parent memory | mitigate | Child creates fresh HttpHelper/Logger instances per RESEARCH.md Pitfall 6. No shared file descriptors for file handles. API keys are in config arrays (innocent data), but fresh instances prevent accidental cross-talk. |
| T-03-06 | Spoofing | Signal inheritance after fork | mitigate | Child immediately resets SIGALRM to SIG_DFL and calls pcntl_alarm(0) after fork (per RESEARCH.md Pitfall 3). This prevents inherited alarm signals from disrupting child execution. |
| T-03-SC | Tampering | Package install risk | accept | No new Composer packages in this phase. phpunit/phpunit already in require-dev from Phase 2. |
</threat_model>
