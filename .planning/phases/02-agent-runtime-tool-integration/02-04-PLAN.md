  <action>
  Apply all six fixes. Each is a narrow surgical change — touch only the identified lines.

  **CR-01 (bin/check-config:20-22):** After the echo on line 21, add `exit(0);` so that when glob() returns false or empty, the script terminates before reaching the foreach on line 24.

  **CR-02 (AcademicSearch.php:260):** Change the URL format string from `'https://api.semanticscholar.org/%s'` to `'https://www.semanticscholar.org/paper/%s'`. This makes the URL resolve to the human-readable paper page instead of the API endpoint.

  **WR-03 (AcademicSearch.php:300-307):** In the deduplicate() method, inside the `if (!empty($paper['abstract']) && empty($existing['abstract']))` block (line 303-304), after `$deduped[$key] = $paper;` add `$seen[$key] = $paper;` to keep the seen cache in sync with the deduped result. This ensures a third duplicate compares against the improved entry.

  **WR-04 (HttpHelper.php:98):** Replace the bare `curl_multi_select($multiHandle, 5);` call with a guarded version:
  ```
  $selRet = curl_multi_select($multiHandle, 5);
  if ($selRet === -1) {
      usleep(10000); // 10ms fallback if select interrupted
  }
  ```
  This prevents busy-wait spinning when curl_multi_select is interrupted by a signal.

  **WR-05 (AcademicSearch.php:20):** Change `'http://export.arxiv.org/api/query'` to `'https://export.arxiv.org/api/query'` — use HTTPS to prevent MITM tampering of paper metadata.

  **WR-06 (LlmClient.php:107-121):** In getLastResponseInfo(), add a null guard before accessing `$this->lastResponse['model']`. The current code accesses `$this->lastResponse['usage']` and `$this->lastResponse['model']` on line 109-110, which emits PHP 8.x Warning "Trying to access array offset on value of type null" if called before a successful chat(). Fix: assign `$response = $this->lastResponse ?? [];` at the start of the method, then access `$response['usage']` and `$response['model']` instead of `$this->lastResponse[...]`.
  </action>
