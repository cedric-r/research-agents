  <behavior>

    - Test: handles Semantic Scholar JSON parsing

  </behavior>
  <action>
    Create src/Tool/AcademicSearch.php in App\Tool namespace per D-09 and D-10, TOOL-03/TOOL-04/TOOL-07:

    Constructor: __construct(HttpHelper $http, array $config = []) where $config is optional:
    - 'max_results' (int, default 5 — per-API limit; total merged = up to 2x this)
    - 'semantic_scholar_api_key' (string, optional — for higher rate limits)

    Method: execute(array $params): string where $params:
    - 'q' (string, required — search query)
    - 'max_results' (int, optional)

    Implementation:

    **arXiv query** (TOOL-03):
    - URL: http://export.arxiv.org/api/query?search_query=all:{urlencode($q)}&start=0&max_results={$max}
    - Call via HttpHelper::get() — arXiv is HTTP (not HTTPS), curl handles redirects with CURLOPT_FOLLOWLOCATION
    - Parse Atom XML with SimpleXMLElement per RESEARCH.md Pattern 3:
      - Register namespaces: atom=http://www.w3.org/2005/Atom, arxiv=http://arxiv.org/schemas/atom
      - foreach ($xml->entry as $entry): extract id, title (clean whitespace), summary (truncate to 300 chars via mb_substr), published (date string), author names, link (alternate rel -> abstract URL), arxiv:doi via $entry->children('http://arxiv.org/schemas/atom')->doi
      - Use trim(preg_replace('/\s+/', ' ', $string)) on title and summary for whitespace normalization (arXiv often includes newlines)

    **Semantic Scholar query** (TOOL-04):
    - URL: https://api.semanticscholar.org/graph/v1/paper/search/bulk?query={urlencode($q)}&limit={$max}&fields=title,abstract,year,authors,citationCount,externalIds&sort=citationCount:desc
    - Call via HttpHelper::get() with headers ['Accept: application/json']
    - Parse JSON: $data['data'] ?? []
    - Extract per paper: title, abstract (truncate 300 chars), year, authors, citationCount, externalIds (DOI, ArXiv)

    **Merge and deduplicate** (D-09, TOOL-07):
    - Collect papers from both APIs into one array
    - Deduplication key: arXiv ID from arXiv entry id URL (extract ID from http://arxiv.org/abs/XXXX.XXXXX) vs Semantic Scholar externalIds['ArXiv']. Also deduplicate by DOI (externalIds['DOI']).
    - Remove duplicates where same arXiv ID or DOI found in both sources (keep the one with more complete data)
    - Sort merged results by year descending, then citationCount descending

    **Format results** (D-10):
    - "## Academic Paper Results\n- {title} ({authors}, {year or 'n.d.'}): {abstract}\n  URL: {url} | Citations: {citationCount} | DOI: {doi}\n---\n..."
    - Authors: join first 3 author names with commas, add "et al." if more than 3
    - Truncate abstract to 300 chars, title to 200 chars

    **Error handling:**
    - arXiv failure: log warning, continue with Semantic Scholar only
    - Semantic Scholar failure: log warning, continue with arXiv only
    - Both fail: return empty string, log error
    - Empty results: return empty string (caller formats as "No results found")
  </action>

  </verify>
