## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PHP process -> external APIs | All external HTTP calls cross this boundary via HttpHelper (LLM, Brave, arXiv, Semantic Scholar) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | HttpHelper::get()/post() | mitigate | JSON_THROW_ON_ERROR for response parsing. curl_exec false check before processing. Never return raw curl error details in user-facing output. |
| T-02-02 | DoS | HttpHelper curl calls | mitigate | CURLOPT_TIMEOUT=60s, CURLOPT_CONNECTTIMEOUT=10s enforced on all calls. Hardcoded minimum — not overridable from user input. |
| T-02-03 | Information Disclosure | HttpHelper error messages | mitigate | Truncate response bodies in error messages (follow D-14 pattern from Phase 1's LlmException). Never include auth headers or full URL query params in error text. |
| T-02-04 | Tampering | ToolRegistry::run() | mitigate | Handler is callable registered by internal code only (never from user input). Params are array — validated by handler, not registry. |
</threat_model>
